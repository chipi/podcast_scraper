## MVP bar (definition of done for v2.6)

An operator can:

1. Enable **GIL** and/or **KG** with documented config; defaults keep today’s behavior
   when flags are off.
2. Run the pipeline and obtain **valid** `*.gi.json` / `*.kg.json` under the output tree.
3. Run **`gi` / `kg` CLI** to validate, inspect, export (NDJSON/merged), and structured
   corpus rollups (**`gi explore` / `gi query`**, **`kg entities` / `kg topics`**) **without
   a database**.
4. Rely on **CI** for schema validation and agreed **enforce** thresholds where already
   wired (must stay green for `make ci-fast` / merge policy).
5. **Demonstrate value** via #445 in the **upcoming** PR; **scientific baselines** via
   issue 443 (and related **448** metrics when picked up) in the **follow-up** PR — without
   blocking on #373 / #446.

## Checklist — add to scope until these are true

### A. Knowledge Graph — core (closes #461)

- [ ] `generate_kg` (or equivalent) **off by default**; independent of GIL.
- [ ] Per-episode **KG artifact** path/name and content aligned with
  [RFC-055](../rfc/RFC-055-knowledge-graph-layer-core.md) / [PRD-019](../prd/PRD-019-knowledge-graph-layer.md).
- [ ] `docs/kg/kg.schema.json` + **`make validate-kg-schema`** (or equivalent) exercised in CI.
- [ ] `docs/kg/ontology.md` aligned with shipped node/edge types.
- [ ] Pipeline integration after prerequisites (e.g. transcript) per RFC-055; module
  boundaries respected.
- [ ] Tests: unit (builders, IDs, validation), integration (fixture → artifact); optional
  E2E acceptable if acceptance or critical-path tests cover the same contract.

### B. Knowledge Graph — consumption (closes #463 / #464)

- [ ] **`kg validate`** — `*.kg.json` vs schema; `--strict` / `--quiet` as documented.
- [ ] **`kg inspect`** — one episode or path; pretty/JSON output.
- [ ] **`kg export`** — NDJSON and merged bundle over `--output-dir` (parity with
  `gi export`).
- [ ] **`kg entities`** — cross-episode roll-up (episodes, mention counts) with documented
  thresholds (`--min-episodes`, etc.).
- [ ] **`kg topics`** — topic pair co-occurrence within episode; `--min-support` documented.
- [ ] **`docs/api/CLI.md`** and **`docs/guides/KNOWLEDGE_GRAPH_GUIDE.md`** updated for all
  shipped subcommands.
- [ ] **`config/acceptance/kg/`** — configs for stable KG paths when acceptance runner is
  in use; E2E or critical-path test that asserts artifacts + strict validation where
  agreed.
- [ ] Merge/export semantics and “logical union over a run tree” **documented** (guide or
  RFC cross-links); SQL insert stubs / **RFC-051** are **not** required for 2.6.

**Defer explicitly (do not block 2.6):** RFC-051/PRD-018 projection, RFC-053 routing,
richer LLM KG extraction, NL `kg query`, KG ↔ GIL `insight_id` links.

### C. Grounded Insights — consumption (closes #439 at deterministic v1)

- [ ] Cross-episode **scan** of output tree; load per-episode `gi.json`; optional strict
  validation behavior documented.
- [ ] **`gi explore`** — topic-led Insight Explorer output per
  [RFC-050](../rfc/RFC-050-grounded-insight-layer-use-cases.md) (insights +
  `supporting_quotes`, counts, `top_speakers` where applicable).
- [ ] **`gi query`** — **UC1–UC5** satisfied via **deterministic pattern mapping**, not
  open-ended NL (matches RFC-050 v1 consumption).
- [ ] CLI flags documented (`--limit`, `--min-confidence`, `--grounded-only`, `--sort`,
  `--format`, `--out`, `--strict`, exit codes) as on #439.
- [ ] **`gi export`** — NDJSON/merged parity with `kg export`.
- [ ] **`gi validate`** — parity story with `kg validate` where applicable.
- [ ] Integration tests on multi-episode fixtures; E2E or doc example if required by
  testing strategy.
- [ ] **`docs/guides/GROUNDED_INSIGHTS_GUIDE.md`** and **`docs/api/CLI.md`** updated.

**Defer:** NL interpretation for arbitrary questions, RFC-051-backed explore. (Interactive
**visualization** for value demos is **in scope** via [#445](https://github.com/chipi/podcast_scraper/issues/445),
not deferred here.)

### D. Quality gates and observability (light)

- [ ] Schema validation and any **enforce** scripts for GI/KG that **CI depends on** are

- [ ] [\#448](https://github.com/chipi/podcast_scraper/issues/448): targeted for the
