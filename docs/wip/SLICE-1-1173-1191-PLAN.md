## #1173 — Transcript segment timestamps drift from audio → **CLOSE**

**State:** the *fix* is landed (`transcription/word_timestamps.py` — `apply_word_timestamps` +
`apply_nested_word_timestamps`, in main via the #1190 squash). What's missing is the **proof**:
the ACs (harness, verification, prod-safety, regression test). This is the whole remaining lift.

**Assets already present:**
- 46 fixture audios `tests/fixtures/audio/v3/*.mp3` (deterministic `say`→aiff→mp3).
- 45 RTTM ground-truth `tests/fixtures/transcripts/v3/*.rttm` (per-turn `start dur`).
- `*.groundtruth.json` (carries `rttm_sha256`, speaker/voice map), `make_groundtruth.py`.

**Plan (repro-first):**
1. **AC1 — drift harness.** Add a scorer that, per fixture: transcribe the mp3 **with word
   timestamps** via the **offline local whisper** path (no paid API — CI/LLM rule), apply the
   refinement, align each transcript segment to the RTTM turn it overlaps, and report
   **start/end drift mean · p95 · max (ms)** across all 46. Emit a JSON report + a summary table.
   Land under `evaluation/` (or `tests/fixtures/scripts/`) so it's re-runnable.
2. **AC2 — verify the target.** Run the harness; confirm **p95 start-drift ≤ 300 ms, max ≤ 1000 ms**.
   If it misses, investigate the two named suspects (chunk-offset accumulation on long audio;
   raw↔ad-free/format-conversion segment rewrite) and fix, then re-verify.
3. **AC3 — prod-safety check.** Ground-truth-free consistency guard over the ~100 prod episodes:
   segment times monotonic non-decreasing, within `[0, audio_duration]`, sane speech-rate; report
   zero violations. Runs against the existing prod corpus (read-only).
4. **AC4 — regression test.** Fixture-based test enforcing AC2, kept **CI-cheap**: pin on
   cached/deterministic transcription output (not a live transcribe) so it fails on drift regress
   without a heavy model run in `ci-fast`.
5. **AC5 — document root cause** (drift = whisper segment-level times vs word-level; refinement is
   the fix) in the transcription guide; `make ci-fast` green.

**Open question inside #1173:** does AC1 transcription run in nightly/tier-3 (heavy) with AC4 the
cheap CI guard? — resolve during build; default to that split.

### Spike findings (2026-07-16, base.en local, robust difflib word-alignment)
- **The fix works.** Naive char-cumulative alignment reported 2–3 s "drift" — that was an
  **alignment artifact**, not audio drift. With difflib word-sequence alignment (tolerant of
  ASR garble/insert/delete), fixture turn-boundary drift collapses to:
  - `p01_e04` (82 s, clean): p95 **47 ms**, max 47.
  - `p08_e01` (271 s): p95 **324 ms**, max 360.
  - `p01_e01` (488 s, asr_garble): p95 **322 ms**, max 424.
- **Residual is a fixture/model artifact, not a pipeline bug.** Signed drift is ~−47 ms (whisper
  *leads* the RTTM boundary); larger model (small.en) did **not** improve it (p95 323). The floor
  is whisper base.en word-timestamp jitter + `say` per-turn silence, both fixture-side. Max is
  comfortably < 1000 ms everywhere; p95 sits right at the 300 ms line on the long fixtures.
- **The origin is PROD data (operator), and the fixtures under-represent it.** Shipped prod-v2
  `.segments.json` carry `start/end/text` but **no words** → drift can't be read from stored data;
  prod-v2 has **no rehosted audio**. Real prod audio exists in `.test_outputs/diar-prod90/audio/`
  (90 acast episodes, 40–48 min). Prod repro = re-transcribe with word timestamps and measure the
  **segment-start vs its own first-word-start delta** (the exact quantity the fix rewrites) — this
  is self-contained (no ground truth) and shows the drift **growing over a 45-min episode**, which
  is the reported bug. TTS-sync improvement idea for tighter fixtures → filed to **#1189**, stay on `say`.

### Decision: where to run the heavy prod transcription
40–48 min episodes × word_timestamps. Local base.en ≈ 10–15 min/episode; DGX whisper (research
slot) is fast + production-grade but GPU-mode-swap is shared-state (needs explicit approval).
**Operator (2026-07-16):** do NOT re-transcribe prod wholesale — **reuse existing artifacts**; if
a real-audio number is needed, run only a **small varied subset on DGX**.

### PROD EVIDENCE from REUSE (no transcription) — the real origin, confirmed
Scanned the **218 shipped `.segments.json`** in `.test_outputs/manual/prod-v2/corpus` (start/end/
text; no words; no rehosted audio):
- **Monotonic ✓, in-bounds ✓** (0 non-monotonic, 0 negative, 0 end<start).
- **Coarse-quantized times** — durations cluster at round values: **2.0s ×13165, 1.0s ×12767,
  3.0s ×6819** — segment times snap to seconds, not word-accurate.
- **3293 physically-impossible segments** (>40 chars/sec) across **164/218 episodes** — e.g.
  `"in the United States."` (21 ch) in **0.2 s** = 105 cps. Human speech tops out ≈ 25 cps.
- **Provider = `tailnet_dgx_whisper`** (prod-v2 run metadata); the #1173 fix landed in **exactly
  that provider** (+ openai + ml). Causal chain closed.

### Slice-1a build (revised, evidence-driven)
1. **AC3 consistency guard** (`evaluation/`, reuse — zero compute): `compute_segment_consistency` /
   `enforce_segment_consistency` over a corpus's `.segments.json` — monotonicity, bounds, and an
   **impossible-cps** metric (>40 cps). Flags the pre-fix prod corpus (164 eps); a re-run with the
   fixed providers clears it.
2. **AC1/AC2 fixture harness** — difflib-aligned turn-boundary drift over the 46 v3 fixtures
   (validated p95 ~300 ms, max <450 ms base.en); report refined-vs-unrefined. Cache words once.
3. **AC4 regression test** — cached words, asserts pooled max ≤ 1000 ms + refined beats unrefined.
4. **DGX small varied subset** (approved) — 3–5 varied `diar-prod90` episodes re-transcribed with
   the fixed DGX provider → coarse→word-accurate; impossible-cps → ~0. The absolute real number.
5. **AC5** — document root cause (coarse segment-level times) + fix in the transcription guide.
